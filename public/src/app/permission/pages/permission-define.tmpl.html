Defining Roles & Permissions

To get started we recommend looking at the example app and how the roles and permissions are defined there.

Take a look at the file permission/permission.run.js to see how the apps roles & permissions.

For example

permission/permission.run.js
// create permissions and add check function verify all permissions var permissions = ['viewEmail', 'viewGitHub', 'viewCalendar', 'viewLayouts', 'viewTodo', 'viewElements', 'viewAuthentication', 'viewCharts', 'viewMaps']; PermissionStore.defineManyPermissions(permissions, function (permissionName) { return UserService.hasPermission(permissionName); }); // create roles for app RoleStore.defineManyRoles({ 'SUPERADMIN': ['viewEmail', 'viewGitHub', 'viewCalendar', 'viewLayouts', 'viewTodo', 'viewElements', 'viewAuthentication', 'viewCharts', 'viewMaps'], 'ADMIN': ['viewLayouts', 'viewTodo', 'viewElements', 'viewAuthentication', 'viewCharts', 'viewMaps'], 'USER': ['viewAuthentication', 'viewCharts', 'viewMaps'], 'ANONYMOUS': [] });

First of all we create a list of permissions then tell permission store about them and assign a function to verify them. We use a service called UserService to check if the current user has a permission.

Next we define the roles SUPERADMIN, ADMIN, USER, and ANONYMOUS. These roles each have their own permissions except for ANONYMOUS which doesn't have any permissions set.

Thats all you need to do to create Roles and Permissions which you can now use to hide / show menu items and page elements.

For more in depth information see the Angular Permissions Module
